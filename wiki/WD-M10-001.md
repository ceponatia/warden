# WD-M10-001: Blocked Trajectory Nodes

## Metric
M10: Trajectory Health

## Summary
The trajectory graph contains nodes with a "blocked" status.

## Description
A "blocked" status indicates that work on a specific task cannot proceed because of external dependencies, unresolved findings, or other blockers. While some blocking is normal in a complex project, a high number of blocked nodes can indicate a breakdown in the delivery pipeline or a lack of clear ownership.

## Causes
- Unresolved high-priority findings.
- Missing external resources or team handoffs.
- Technical debt preventing implementation.

## Resolution
- Review the blockers for each affected node.
- Reassign or escalate tasks that have been blocked for more than 7 days.
- Use `warden trajectory patch` to update status once dependencies are cleared.
