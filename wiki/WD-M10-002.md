# WD-M10-002: Stale Trajectory Graph

## Metric
M10: Trajectory Health

## Summary
The repository's trajectory graph hasn't been updated recently.

## Description
The trajectory graph is intended to be a living map of the project's coding path. If it isn't updated for a long period (default 10 days), it no longer accurately reflects the project's state or the team's "vibe." This staleness can lead to agent drift and misalignment between humans and AI.

## Causes
- Project inactivity.
- Shift to manual task management outside of the trajectory system.
- Failure to run `warden collect` or `warden trajectory import` after significant changes.

## Resolution
- Run `warden trajectory validate` to check current state.
- Update the trajectory via the CLI, Dashboard, or MCP.
- Consider reducing the `staleDays` threshold if more frequent updates are desired.
